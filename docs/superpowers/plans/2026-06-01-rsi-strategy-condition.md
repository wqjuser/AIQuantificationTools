# RSI 策略条件贯通计划

## 背景

P0 策略工坊目标要求基础规则不只停留在 SMA 和成交量确认，还要覆盖 RSI 等常用动量/反转指标。本轮把 RSI 从文本草稿、策略校验、回测执行到前端审计回放全部打通，避免 UI 显示支持但回测引擎不识别。

## 用户任务

用户在 Strategy Lab 输入或载入 `RSI14 < 30` / `RSI14 > 55` 这类规则后，可以运行研究流水线，生成可复现回测，并在前端看到结构化 RSI 条件、参数标签和策略就绪闸门。

## 范围

- 后端新增 RSI 指标计算函数。
- `StrategyConfig` 支持 `rsi_below` 和 `rsi_above` 条件。
- 研究快照解析 RSI 条件时不能误补默认 SMA 条件。
- 回测引擎按 RSI 阈值触发入场/出场。
- 策略预检接受结构化 RSI 条件并显示可读 gate value。
- 前端审计回放和规则矩阵能格式化 RSI 条件。
- 更新产品规划和架构说明。

## 非目标

- 本轮不做可视化 RSI 表单控件。
- 本轮不做图表 RSI 副图绘制。
- 本轮不做组合级 RSI 策略或多指标表达式编辑器。

## 测试计划

- 后端解析测试：`RSI14 < 30` 生成 `rsi_below(window=14, threshold=30)`，且不会默认追加 SMA。
- 后端回测测试：固定下跌样例数据只有 RSI 进入阈值后才入场。
- 后端预检测试：纯 RSI 入场/出场策略通过 schema gate。
- 前端模型测试：审计 run 中的 RSI 条件恢复为 `RSI14 < 30` / `RSI14 > 55`，规则矩阵参数显示 `RSI14<30` / `RSI14>55`。
- 全量验证：`npm test`、`npm run build`、`git diff --check`。
