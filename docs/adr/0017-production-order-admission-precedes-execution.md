# 生产委托准入准备先于生产执行

Stage 8 之后的下一阶段只交付生产委托准入准备，并继续禁止创建、查询、撤销或同步生产委托。该阶段只面向单一 Binance Spot 专用现货账户并复用 Stage 7 生产只读凭据，不覆盖 Futures、Margin、多账户或其它交易所，也不读取、保存或探测已开启交易权限的生产 Key；只读凭据可以判断候选资金是否充足，但审计仅保留布尔结论和观测时间，不保存资产名称或余额。准入结论必须绑定单个权威工作流、同一工作流的 Stage 6 已成功对账终态批次、当前 Stage 8 生产只读连续性和全部精确订单内容哈希，不能由手填订单或全局退出清单替代，也不能成为账户级许可或全局实盘开关；每个候选只允许一次不可改写的具名人工复核，复核通过仍不产生生产执行权。准入急停直接复用 Stage 8 持久化 `revoke / restore`，本阶段不创建没有生产订单可撤的第二套 kill switch；候选和复核可以进入研究包，但导入副本只允许校验与回放。生产只读到真实资金写操作是独立安全边界，只有后续单独完成人工授权与验收，才可以引入生产交易凭据并讨论受限生产试单，避免把“准备就绪”误当成实盘许可。
